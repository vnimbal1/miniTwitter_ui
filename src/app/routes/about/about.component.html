Welcome to Mini Twitter Application

An endpoint to read the message list for the current user (as identified by their HTTP Basic authentication credentials). Include messages they have sent and messages sent by users they follow. Support a “search=” parameter that can be used to further filter messages based on keyword.
http://localhost:8080/people/messages
http://localhost:8080/people/messages?search=keyword
Messages are ordered by user id
Endpoint to get the list of people the user is following
http://localhost:8080/people/following
Endpoint to get the list of followers of the user
http://localhost:8080/people/followers
An endpoint to start following another user
http://localhost:8080/people/follow/ userId
Note: userId of user to be followed
An endpoint to unfollow another user
http://localhost:8080/people/unfollow/ userId
Note: userId of user to be unfollowed

Additional functionality

1. An endpoint that returns the current user's "shortest distance" to some other user. The shortest distance is defined as the number of hops needed to reach a user through the users you are following (not through your followers; direction matters). For example, if you follow user B, your shortest distance to B is 1. If you do not follow user B, but you do follow user C who follows user B, your shortest distance to B is 2.
http://localhost:8080/people/distance/ userId
Distance to User 10 from current user
Note: userId of user for which we need to find distance
Distance of -1 indicates no connection
In target is same as user that is logged in then distance is 0
2. An endpoint that returns a list of all users, paired with their most "popular" follower. The more followers someone has, the more "popular" they are. Hint: this is possible to do with a single SQL query!
http://localhost:8080/people/popular
